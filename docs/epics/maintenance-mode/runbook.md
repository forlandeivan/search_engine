# Runbook: ручное выключение режима обслуживания (MVP)

## Назначение
Инструкция для выключения режима обслуживания, если администратор не может войти в систему.

## Когда использовать
- режим обслуживания активен, а вход в админку недоступен;
- требуется срочно вернуть доступ пользователям;
- плановое обслуживание завершено, но статус не снят.

## Предварительная проверка
1. Убедиться, что режим активен: `GET /api/maintenance/status`.
2. Зафиксировать текущие значения настроек (на случай отката).

## Действия
1. Получить доступ к хранилищу настроек режима обслуживания.
2. Найти запись `maintenance_mode_settings` с `id = maintenance_mode_singleton`.
3. Установить значения:
   - `force_enabled = false`
   - `scheduled_start_at = null`
   - `scheduled_end_at = null`
   - `public_eta = null` (если было задано)
   - `message_title = ""`
   - `message_body = ""`
4. Обновить `updated_at` на текущее время.
5. Если записи нет — создать её с указанными значениями.

## Проверка результата
1. Повторить `GET /api/maintenance/status` — ожидается `status = "off"`.
2. Открыть основное приложение — экран обслуживания отсутствует, вход доступен.

## Примечания
- Ручное изменение не попадёт в аудит‑журнал. Зафиксируйте действие в операционном логе.
- Если статус не сменился, проверьте кэш и время на сервере.
