# Нефункциональные требования: режим обслуживания (MVP)

## Производительность
- Проверка режима обслуживания добавляет минимальную задержку (≤ 5 мс на запрос).
- Статус кэшируется на 30–60 секунд.

## Отказоустойчивость
- Если источник конфигурации недоступен, система переходит в безопасный режим:
  - показывается экран обслуживания;
  - вход блокируется.

## Безопасность
- Изменение режима доступно только `admin`.
- Все попытки изменить режим без прав фиксируются в логах.

## Масштабируемость
- Статус режима консистентен между инстансами (при Redis — общий кэш).

## Расширяемость
- В дальнейшем возможны:
  - частичная доступность (read‑only);
  - пер‑воркспейс обслуживание.
