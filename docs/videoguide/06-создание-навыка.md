# Раздел 6: Создание навыка с поиском по базе знаний

**Хронометраж:** 4-5 минут

---

## Текст для озвучки

Навык — это настроенный искусственный интеллект с определёнными способностями. Навык определяет, как система обрабатывает ваши запросы, какие документы использует и в каком стиле отвечает.

Переходим в раздел "Навыки" и нажимаем кнопку создания нового навыка.

В поле названия вводим понятное имя. Например, "Помощник по судебной практике".

В поле описания кратко укажите назначение навыка. Это поможет вам и коллегам понимать, для чего он создан.

Выберите иконку для навыка. Это визуальный идентификатор, который помогает быстро находить нужный навык в списке.

Теперь подключаем базу знаний. Нажмите на поле выбора баз знаний и выберите созданную ранее базу "Судебная практика по гражданским делам". Вы можете подключить несколько баз одновременно, если хотите искать по разным категориям документов.

Выберите модель искусственного интеллекта для генерации ответов. Рекомендую использовать модель, указанную по умолчанию, если у вас нет особых предпочтений.

Обратите внимание на параметр "Показывать источники". Рекомендую оставить его включённым. Благодаря этому вы будете видеть, на какие документы опирается ответ, и сможете проверить информацию.

В настройках также есть параметр истории сообщений — это контекст предыдущего диалога, который система учитывает при формировании ответа.

При желании настройте системный промпт. Это инструкция для искусственного интеллекта, определяющая стиль и формат ответов. Например, можно указать: "Ты — помощник судьи. Отвечай чётко и по существу, ссылаясь на конкретные судебные акты."

Нажимаем кнопку "Создать", чтобы сохранить навык.

Навык создан и готов к использованию. Перейдём к работе с чатом.

---

## Что показывать на экране

1. Переход в раздел "Навыки"
2. Список существующих навыков (если есть)
3. Нажатие кнопки "Создать навык"
4. Форма создания навыка:
   - Поле "Название" — ввод "Помощник по судебной практике"
   - Поле "Описание" — краткий текст
   - Выбор иконки (IconPicker)
5. Подключение базы знаний:
   - Клик на мультиселект (KnowledgeBaseMultiSelect)
   - Выбор базы "Судебная практика по гражданским делам"
   - Бейдж выбранной базы
6. Выбор модели LLM (выпадающий список)
7. Чекбокс "Показывать источники" — убедиться что включён
8. Параметр истории сообщений (контекст диалога)
9. Поле "Системный промпт" — ввод инструкции (опционально)
10. Нажатие кнопки "Создать"
11. Навык появляется в списке
