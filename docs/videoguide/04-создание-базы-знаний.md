# Раздел 4: Создание базы знаний и импорт документов

**Хронометраж:** 5-6 минут

---

## Текст для озвучки

База знаний — это структурированное хранилище документов, по которым искусственный интеллект будет осуществлять поиск. Давайте создадим базу и загрузим в неё документы судебной практики.

Переходим в раздел "Базы знаний" и нажимаем кнопку создания новой базы.

Система предлагает несколько способов.

Первый способ — пустая база. Подходит, когда вы планируете добавлять документы вручную по одному.

Второй способ — импорт архива. Позволяет загрузить ZIP-архив с документами, сохраняя структуру папок.

Третий способ — краулинг сайта. Автоматически собирает документы с указанного веб-сайта.

Четвёртый способ — импорт джейсон документа. Подходит для загрузки структурированных данных в формате джейсон или джейсон-лайнс.

Для нашего примера мы используем датасет судебных решений из открытых источников и он как раз в таком формате.

Выбираем "Импорт джейсон". В появившемся окне вводим название базы знаний. Назовём её "Судебная практика". При желании добавьте краткое описание.

Теперь загружаем файл. Нажмите на область загрузки или перетащите файл в это поле.

После загрузки система анализирует структуру файла и показывает предварительный просмотр. Здесь вы видите, какие поля содержит ваш файл и сколько записей будет импортировано.

На следующем шаге настраиваем маппинг полей. Система должна понимать, какое поле содержит заголовок документа, какое — основной текст, а какие — дополнительные метаданные.

Укажите поле с заголовком документа. Обычно это название дела или номер судебного акта.

Однако в нашем датасете не представлены такие данные, поэтому мы применим автогенерацию заголовка документа при помощи искусственного интеллекта. В момент импорта он будет создан на основе нашей инструкции и краткого содержания документа из поля "текст"

Укажите поле с основным содержимым.

При необходимости настройте дополнительные поля — категорию дела и другие атрибуты.

На этапе настройки иерархии вы можете указать, как группировать документы. Например, по категориям дел. Если иерархия не нужна, пропустите этот шаг.

Нажимаем кнопку "Создать" для запуска импорта. Система начнёт обработку файла. Длительность зависит от объёма данных. Для нескольких сотен документов это занимает около минуты, но с учетом AI-генерации заголовка может занять больше времени.

После завершения импорта вы увидите базу знаний со всеми загруженными документами. Можно открыть любой документ и проверить, что содержимое импортировано корректно.

Документы загружены, но искусственный интеллект пока не может по ним искать. Для этого необходимо выполнить индексацию.

---

## Что показывать на экране

1. Переход в раздел "Базы знаний"
2. Нажатие кнопки создания новой базы
3. Диалог выбора способа создания — показать все 4 варианта:
   - Пустая база
   - Импорт архива
   - Краулинг сайта
   - Импорт JSON
4. Выбор "Импорт JSON"
5. Ввод названия: "Судебная практика по гражданским делам"
6. Область загрузки файла — перетаскивание JSON-файла
7. Предварительный просмотр структуры (поля, количество записей)
8. Шаг маппинга полей:
   - Выбор поля заголовка
   - Выбор поля содержимого
   - Дополнительные поля (опционально)
9. Шаг настройки иерархии (можно пропустить)
10. Нажатие кнопки "Создать"
11. Прогресс импорта
12. Готовая база знаний с документами
13. Открытие одного документа для проверки
