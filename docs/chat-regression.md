## Баг с фантомным bubble (только время, без текста) — регрессия/проверки

### Что было
- После отправки пользовательского сообщения создавался локальный ассистентский placeholder с пустым `content`.
- Если стрим или серверный ответ не успевал/не заменял его, он оставался в `localMessages` и рендерился как пустой bubble (только время).
- После перезагрузки пустой bubble исчезал, потому что на бэкенде такого сообщения нет.

### Что изменили
- В `ChatPage.streamMessage` временный ассистентский placeholder удаляется при `onDone` и при `onError`.
- В `ChatMessagesArea` при рендере отсекаются локальные ассистенты без `content` (id начинается с `local-assistant`).

### E2E / интеграционные проверки
Если есть e2e (Playwright/Cypress):
1. Открыть чат в workspace.
2. Отправить текстовое сообщение.
3. Дождаться ответа ассистента (стрим или финальный).
4. Проверить, что в истории только 1 user bubble и 1 assistant bubble, нет пустых bubble (по селектору/отсутствию пустых блоков).
5. Сделать reload страницы, снова убедиться, что пустых bubble нет.

Если e2e нет — интеграционный тест:
- Смонтировать контейнер чата с mock-данными.
- Симулировать отправку (добавить локальный `local-assistant`), затем «серверный» ответ.
- Убедиться, что после `onDone` нет пустых ассистентских сообщений в рендере/стейте.

### Чек-лист для ручного теста
- Отправка одного сообщения → ассистент ответил → нет пустых bubble.
- Несколько сообщений подряд → порядок и количество bubble корректны.
- Медленный стриминг: ассистент «печатает», после завершения нет пустых bubble.
- Ошибка отправки (например, отключить сеть) → нет пустого ассистента, placeholder исчезает при ошибке.
- Обновить страницу в любой стадии (до/после ответа) → после загрузки нет пустых bubble.
