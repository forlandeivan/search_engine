Коротко, что есть в “холсте” справа (при клике на «Открыть стенограмму»):

Загрузка и отображение: по transcriptId дергается GET /api/workspaces/:workspaceId/transcripts/:transcriptId, в правой колонке показывается полный текст стенограммы. Открывается по клику на сообщение с metadata.type="transcript".
Разметка/раскладка: холст не модалка; чат остаётся слева, стенограмма — справа (двухколоночный режим, иногда получается трехколоночный, если открыт список диалогов).
Состояния:
processing — показывается текст «Идёт расшифровка аудио…» со спиннером.
ready — текст стенограммы с кнопкой «Открыть стенограмму» (по клику — сам холст).
failed — сообщение об ошибке («Не удалось распознать аудио…»).
Редактирование (было добавлено): холст в режиме редактора (draftText), с кнопкой «Сохранить» (отправляет PUT /api/workspaces/:workspaceId/transcripts/:transcriptId с fullText) и «Сбросить» (возвращает к последней версии с бэка). Иконки/тулбар — простые (textarea + мини тултип по полям Target/Input/Output).
Выделение/статусы: карточка в чате несёт metadata.transcriptStatus и previewText; при ready можно открыть холст, при processing — только плейсхолдер.
Сортировка/открытие: открывается по id из сообщения; после сохранения текст обновляется и при следующем открытии заново подтягивается с бэка.
Для чего: это основной экран работы со стенограммой — читать полный текст, при необходимости подправить и сохранить, не превращая транскрипт в «обычные» сообщения чата.

а что насчет действия из навыков, они там есть?


Да. В холсте справа подтягиваем и показываем действия навыка:

Берём /skills/:skillId/actions, фильтруем enabled, enabledPlacements содержит canvas, target обычно transcript/selection.
Отображаем кнопками в блоке «Действия» справа; при клике отправляем /skills/:skillId/actions/:actionId/run с placement=canvas, target/selection и applyMode=apply.
По успешному выполнению рефетчим/обновляем стенограмму; для ошибок показываем статус/тост.
Если у навыка нет включённых canvas‑действий, блок пуст.
